#对话流

##基本的对话结构

对于大多数的功能，对话主体采用简单的格式化文本进行配置。格式化文本由编号、发言人、对话文本（或url）三部分组成，使用英文双冒号（::）进行分离。在填充完毕后会自动识别网址为url，同时会指定跳转至下一条对话。音频部分可省略。
其基本结构为如下状况：
'（对话序号）::（发言人）::（对话文本/打开网页url）::（音频url，此为可选项）',
对话序号仅限英文字母、数字、减号（-）以及下划线（_），使用其他符号可能会导致未知错误。这不是对话id，仅仅是一个缩略的序号。在保证无重复的情况下应尽量简短，以方便检视。
发言人为剧本文本中指定的发言人，可能与模型名不同。发言人可以使用英文字母、汉字、数字、各种全角符号、减号（-）、下划线（_）乃至空格（ ）等，但不得完全留空，不可使用井号（#）以及英文冒号（:）。留空或使用禁止的字符会导致未知错误。这些风险应当在剧本编写时规避，此处仅为重复强调。
对话文本为剧本文本中指定的文本，而网页url则为剧本中指定打开的网页。其内容完全由剧本文本决定。在特殊符号的使用方面应当在剧本中注意，不做额外说明。
在基本对话结构之外，还可以插入许多额外的附加指令，它们插入的位置分别有不同要求，具体位置要求参见后续段落的说明。下面是几个不包含指令的对话的基本样例。

###样例1：
剧本文本：
小灵语：你好，我是小灵语。

格式化对话：
'0::小灵语::你好，我是小灵语。',

样例说明：
这个对话只有发言人以及对话文本。

###样例2：
剧本文本：
小灵语：你好，我是小灵语。
小灵语：欢迎来到“灵镜新世界”。

格式化对话：
'0::小灵语::你好，我是小灵语。',
'1::小灵语::欢迎来到“灵镜新世界”。',

样例说明：
这个对话有发言人以及对话文本，这因为是连续两句所以按照顺序格式化，并使用了不同的序号。

###样例3：
剧本文本：
小灵语：（打开网址：https://h5.zspiritx.com.cn/processh5/guide?story_id=7）

格式化对话：
'2::小灵语::https://h5.zspiritx.com.cn/processh5/guide?story_id=7',

样例说明：
这个对话只有发言人以及网页url。

##nextID与userSelections

可以注意到，上面的三个对话都没有指出“下一条对话”是什么。在默认情况下，一条对话执行完毕后，之后会执行下一条对话，这是默认的线性阅读逻辑。然而在一些特殊情况下，我们需要在一行对话后返回之前的某一行，或跳跃到之后的某一行，此时我们就需要专门指定执行目标。nextID指令用于指出执行完此行对话时跳转的目标。nextID指令需要标注于对话的对话序号段（即，第一个双冒号（::）之前），其结构如下：
{#nextID#（这里是跳转对话的对话序号，使用#分开）}
此处的对话序号需要与目标行的对话序号完全一致，无需auto标识符。

可以注意到的是，nextID可以指向多个不同的对话，这是因为除去顺序执行对话，在一些情况下用户可能通过网页返回参数或文本选项执行多种不同的下一句。对于网页返回参数，网页会执行相应的处理，在对话信息中只需按顺序写好跳转目标即可；而文本选项则需要通过userSelections来列出选项，选项内容与跳转目标在nextID中的顺序需要一致。userSelections指令需要在对话的文本或网页的url段（即，第二个双冒号（::）之后，第三个双冒号（::）（若有）之前），其结构如下：
{#userSelections#（这里各个选项的文本，使用#分开）}
选项文本为剧本文本中指定的文本，其内容完全由剧本文本决定。在特殊符号的使用方面应当在剧本中注意，不做额外说明。

下面是几个与nextID跳转相关的样例。

###样例4：
剧本文本：
白马：接下来我们去哪里？（选项：山上、河边）
选择山上的白马多一句：说实话我有点渴了，要不我们先去喝水吧。
白马：清澈的河水我来了！

格式化对话：
'10{#nextID#11#12}::白马::接下来我们去哪里？{#userSelections#山上#河边}',
'11::白马::说实话我有点渴了，要不我们先去喝水吧。',
'12::白马::清澈的河水我来了！',

样例说明：
这是一个典型的P型对话分支，第一句两个选项中，有一歌选项会跳过后续的一句。
在第一句话中，我们使用userSelections指令展示两个不同选项，并且在nextID中给出各自的跳转目标。
在第二句，这是去山上的剧情分支，因为这一句只是凭空增加的，它后续会回到共同的剧情，所以不额外指定nextID。
在第三句，从此往后是共通剧情，一切如同正常的对话流。

###样例5：
剧本文本：
弓箭手：你选择让谁来开门？（选项：魔法师、战士）
选择魔法师：十三重天的精灵啊，我以火焰之子的身份向你们发出号令！
魔法师：炎爆术！
选择战士：（用力一踢）
无论选了什么都回到这里：弓箭手：看来这门很脆弱，一下就开了。

格式化对话：
'30{#nextID#31#33}::弓箭手::你选择让谁来开门？{#userSelections#魔法师#战士}',
'31::魔法师::十三重天的精灵啊，我以火焰之子的身份向你们发出号令！',
'32{#nextID#33}::魔法师::炎爆术！',
'33::战士::（用力一踢）',
'34::弓箭手::看来这门很脆弱，一下就开了。',

样例说明：
这是一个典型的菱形对话分支，第一句两个选项进入两个不同分支，随后两个分支在后续再次合并。
在第一句话中，我们使用userSelections指令展示两个不同选项，并且在nextID中给出各自的跳转目标。
在第二句，选择魔法师的剧情分支，这只是第一句话，所以顺序向后。
在第三句，选择魔法师的剧情分支结束，需要跳转至此选择的共通结局，所以用nextID跳过河边相关的文本。
在第四句，此行是选择战士的剧情分支，从第一句跳转过来的。同时因为它下面就是共通结局，所以虽然选择战士分支在此结束，它也不需要额外的跳跃指令即可继续执行。
第五句，这是共通结局。

###样例6：
剧本文本：
小仙子：接受我的祝福吧！（选项：好啊、不了）
选择拒绝：再考虑一下吧！真的很有用！（返回选择）
选择同意：我就知道你会同意的！

格式化对话：
'17{#nextID#19#18}::小仙子::接受我的祝福吧！{#userSelections#好啊#不了}',
'18{#nextID#17}::小仙子::再考虑一下吧！真的很有用！',
'19::小仙子::我就知道你会同意的！',

样例说明：
这是一个典型的死循环跳出，第一句两个选项中，有一个选项会将对话卡入死循环，另一个则是跳出。
在第一句话中，我们使用userSelections指令展示两个不同选项，并且在nextID中给出各自的跳转目标。需要额外注意的是，为了让“好啊”这个选项在文本中显示在前面，在nextID中专门颠倒了对话序号的顺序。
在第二句，拒绝的死循环剧情分支，我们按照说明通过指定nextID让它返回选择分歧。
在第三句，从此往后是继续往后剧情，一切如同正常的对话流。

###样例7：
剧本文本：
小灵语：（打开网址：https://h5.zspiritx.com.cn/qa5/quiz?quiz_id=138）
结果1，小灵语：你选择了结果1呢……
结果2，小灵语：你选择了结果2呢……
结果3，小灵语：你选择了结果3呢……
小灵语：很好，我知道你的选择了。

格式化对话：
'0{#nextID#1#2#3}::小灵语::https://h5.zspiritx.com.cn/qa5/quiz?quiz_id=138',
'1{#nextID#4}::小灵语::你选择了结果1呢……',
'2{#nextID#4}::小灵语::你选择了结果2呢……',
'3{#nextID#4}::小灵语::你选择了结果3呢……',
'4::小灵语::很好，我知道你的选择了。',

样例说明：
这同样是一个菱形对话分支，但是这是通过url返回参数而非userSelections指令控制的，所以只需要做好nextID上的映射管理。

##段落结束的空对话

对话并非总是持续连续的，对话段落会以一个空对话结束，其结构非常简单：
'（对话序号）::（跳转对话的对话序号，可省略）',
空对话有一个非常特殊的情况——因为绝大多数空对话需要指向特定旧对话以构造死循环，所以空对话可以在第二分段中写入目标跳转对话的对话序号。其书写方法与使用nextID指令等价，这是一种方便快捷的语法糖。
下面是几个与空对话相关的样例。

###样例8：
剧本文本：
木桩：我是一个木桩。
（结束对话，重复）

格式化对话：
'0::木桩::我是一个木桩。',
'end1::0',

样例说明：
这是一个最简单的对话循环——只有一句话，然后结束对话重复这一句。

###样例9：
剧本文本：
白马：我们要走了，再见！
村民：再见！
白马：接下来，就要靠我们自己了。
（结束对话，重复最后一句）
白马：看，前面有一个山洞，我们在这里过夜吧！
（结束对话，重复）

格式化对话：
'0::白马::我们要走了，再见！',
'1::村民::再见！',
'2::白马::接下来，就要靠我们自己了。',
'end1::2',
'3::白马::看，前面有一个山洞，我们在这里过夜吧！',
'end2::3',

样例说明：
在这个对话样例中，这里是两个先后发生的独立对话串。
在第一个对话串结束后，空对话指向最后一句构成一个死循环。从对话链条来说，这是为了避免在自由对话下触发后续的内容。
在满足特定外部条件后，对话才会从end1跳转至3，此后会在第二个对话串中形成死循环重复。
这种结构常见于拥有大量交互的角色上，这样一个角色可以在剧情进程中的多处分别说出不同的对话，而具体进度则由外部事件控制。

###样例10：
剧本文本：
小灵语（介绍）：（打开网址：https://h5.zspiritx.com.cn/processh5/guide?story_id=7）
（结束对话）
小灵语：需要我再次介绍一下么？（选项：是的，取消）

格式化对话：
'0::小灵语::https://h5.zspiritx.com.cn/processh5/guide?story_id=7',
'end1',
'1{#nextID#0#end1}::小灵语::需要我再次介绍一下么？{#userSelections#是的#取消}',

样例说明：
在这个对话样例中，空对话是继续向后执行的，所以它没有书写任何指向。
在这个对话样例中，最特殊的表现在于后面的选择，其选择支有两个方向：1、重复第一句的介绍；2、取消对话。
关于重复介绍，是很简单的回到介绍句；而取消对话，这里选择的方法是跳转至最近的上一个空对话。
在这个对话样例中，虽然最后一句并不是空对话，然而它的所有结果都是指向既有的对话，所以它依然是一个完整的对话流。

##对话语音

在由文本构成的对话中，有时会有配套的音频资源需要播放。此时应当在对话的文本后加入音频资源指令：
{#sentenceClip#（这里是音频资源的文件地址，一般以 voice/ 开头）} 
音频文件url并非完整url，剧本中标注的音频文件地址一般 voice/ 开始，在整理文件时直接照抄即可，无需做出更改。

###样例11：
剧本文本：
小灵语：欢迎来到“灵镜新世界”，这里您将体验一个全新的AR世界。（音频：voice/guide/Fairy1.mp3）

格式化对话：
'3::小灵语::欢迎来到“灵镜新世界”，这里您将体验一个全新的AR世界。{#sentenceClip#voice/guide/Fairy1.mp3}',

样例说明：
音频文件的地址信息直接通过音频资源指令附加于显示的文本之后。

###样例12：
剧本文本：
小火龙：看来，这就是最后的选择了，红水晶还是蓝水晶呢？（选项：红水晶、蓝水晶）（音频：voice/10/3/aoiubasog.mp3）

格式化对话：
'57{#nextID#58#59}::小火龙::看来，这就是最后的选择了，红水晶还是蓝水晶呢？{#userSelections#红水晶#蓝水晶}{#sentenceClip#voice/guide/Fairy1.mp3}',

样例说明：
这个对话中同时需要附加音频文件与分支选项，音频文件的地址信息直接通过音频资源指令附加于显示的文本之后，而选项则通过userSelections指令与nextID指令分别处理。所有的附加内容只需要出现在对应分段（序号段、文本内容段）即可，顺序不重要。

##附加指令

当对话中需要执行隐藏模型、显示模型、外部控制模型对话跳转这三项常用指令时，在对话的文本或网页的url后，或空对话的最后加入对应的操作指令：
show指令：{#showModels#（这里是需要显示的模型的id，多个模型使用#分开）} 
例子：{#showModels#test-obj1} 
hide指令：{#hideModels#（这里是需要隐藏的模型的id，多个模型使用#分开）} 
例子：{#hideModels#test-obj2} 
entityAction指令：{#entityAction#（这里是需要跳转的指令对，以“对话id缩写@模型id”的结构书写，多个模型使用#分开）}
例子：{#entityAction#5@test-obj3}'

比较特别的，如果需要隐藏模型自身，可以使用{#hideModels#{#this}}这个固定写法，{#this} 等价于模型自己的id，这是一个通用写法，可以避免反复配置。

###样例13：
剧本文本：
魔法师：出现吧！奥林匹斯的大门！（显示模型Gate-Olympic）

格式化对话：
'4::魔法师::出现吧！奥林匹斯的大门！{#showModels#Gate-Olympic}',

样例说明：
因为只有一个模型，直接使用showModels指令即可。

###样例14：
剧本文本：
魔王：哈哈哈！你不会以为这些封印道具对我有用吧！（隐藏模型Germs1、Germs2、Germs3）

格式化对话：
'5::魔王::哈哈哈！你不会以为这些封印道具对我有用吧！{#hideModels#Germs1#Germs2#Germs3}',

样例说明：
因为虽然一次性对三个模型执行了隐藏，但是直接使用hideModels指令并使用井号（#）分隔即可。

###样例15：
剧本文本：
女神：接下来，我将我残存的力量化作弓箭，请好好使用这份力量，来拯救世界吧！（显示模型Bow，隐藏自己）

格式化对话：
'11::女神::接下来，我将我残存的力量化作弓箭，请好好使用这份力量，来拯救世界吧！{#showModels#Bow}{#hideModels#{#this}}',

样例说明：
此处有两个特殊之处——一方面，这句话同时显示与隐藏，所以同时使用showModels与hideModels指令；另一方面，因为隐藏的就是这个模型本身，所以使用的是{#this}这个缩略指令。

###样例16：
剧本文本：
村长：如此，我们就得救了。（模型Park-Angel对话跳跃到17）

格式化对话：
'6::村长::如此，我们就得救了。{#entityAction#17@Park-Angel}',

样例说明：
这是在要求一个模型id为Park-Angel的模型将对话跳转至17，所以使用entityAction指令。

#完整的应用实例
下面是一个完整的复杂结构对话例子：

首次点击小灵语开始对话：你好，我是小灵语。
小灵语：欢迎来到“灵镜新世界”，这里您将体验一个全新的AR世界。（语音文件：voice/guide/Fairy1.mp3）
打开介绍网址：https://h5.zspiritx.com.cn/processh5/guide?story_id=7'
（暂停对话，等待再次点击）
任意时刻点击小灵语：有什么需要帮助的地方？
-选项1：查看教程（跳转至打开介绍网址）
-选项2：取消（返回空闲）

点击小灵语：快去看看新模型吧（显示模型Test-obj1）
小灵语：模型现在收起来了（隐藏模型Test-obj1）
  （结束对话，重复最后两句）
在这个例子中，dilalog0、1、2是首次点击后顺序播放的，随后进入空行的end1；当用户抵达end1后再次对话，会进入对话4的选择分支，用户可以选择再次查看对话2，也可以选择回到end1；对话5、6与end2的循环，则需要在外部通过其他模型的跳转对话进入，可以注意到对话5将一个模型放出了，随后对话6又收回了这个模型，这里仅仅作为代码演示，实际使用中请不要放下模型立即收回。

该例子配置后显示如下：
 array (
  'Name' => '小灵语',
  'Intro' => 'auto-0',
  'ActionOnPlaced' => 
  array (
  ),
  'Dialog' => 
  array (
'0::小灵语::你好，我是小灵语',
'1::小灵语::欢迎来到“灵镜新世界”，这里您将体验一个全新的AR世界::voice/guide/Fairy1.mp3',
'2::小灵语::https://h5.zspiritx.com.cn/processh5/guide?story_id=7',
'end1::4',
'4{#nextID#2#end1}::小灵语::有什么需要帮助的地方？{#userSelections#查看教程#取消}',
'5::小灵语::快去看看新模型吧{#showModels#Test-obj1}',
'6::小灵语::模型现在收起来了{#hideModels#Test-obj1}',
'end2::5',
  ),
);
